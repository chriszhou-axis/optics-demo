# M12 Dome Camera Focus Demo

[English Documentation](./README.md)

这是一个纯前端、无依赖的交互式光学演示，用来观察 M12 广角镜头在球罩前盖存在时，对成像清晰度、焦点位置、传感器模糊和温漂的影响。

当前演示基于固定传感器平面加薄透镜近似模型，面向相机场景，而不是通用放大镜示意。界面支持中文和英文切换。

## 1. 当前镜头参数

- 标称焦距：2.8 mm
- 光圈：F2.0
- 视场角：99° 水平 / 74° 垂直
- 最近对焦距离：1 m
- 接口：M12，固定光圈
- 像素尺寸：2.0 um

## 2. 当前功能

- 在主画布中显示目标物、镜头、球罩前盖、折射光线、像面和传感器平面
- 支持拖动目标物，观察不同物距下的成像变化
- 支持拖动镜头，观察对焦距离与焦平面移动
- 提供传感器预览窗口，用更明显的方式显示模糊和清晰变化
- 支持镜头跟随鼠标开关与跟随强度调节
- 支持镜头温度变化带来的热漂移焦移
- 区分“镜头调校”和“真实 Dome 影响”两类参数
- 支持中英文界面切换
- 实时显示焦距、焦移、模糊圈、清晰度等读数

## 3. 页面结构

页面分为左侧演示区和右侧控制区。

左侧主画布包含以下内容：

- Subject：被观察目标
- Refracted rays：穿过镜头并受 dome 影响后的示意光线
- Dome cover：镜头前方的球罩前盖
- Image plane：当前物距对应的实际像面
- Sensor plane：固定不动的传感器位置
- Sensor preview：右上角放大的传感器预览，用于更清楚地查看虚化程度
- Focus travel：底部对焦行程条，表示当前对焦位置在可调范围中的比例

右侧控制区分为 5 个部分：

- Camera Lens：固定镜头规格信息
- Lens Tuning：镜头调校参数
- Real Dome Influence：真实球罩影响参数
- Interaction：交互行为与重置按钮
- Live Readout：实时计算结果

## 4. 控件说明

### 4.1 Camera Lens

这一栏展示固定规格信息，不参与交互调节。

### 4.2 Lens Tuning

- Lens power bias：调整镜头等效光焦度偏置，影响等效焦距与对焦位置
- Lens aberration bias：调整镜头像差偏置，影响离轴模糊趋势
- Lens temperature：模拟温度变化带来的传感器等效焦移

### 4.3 Real Dome Influence

- Dome tilt：模拟球罩安装倾斜，引入光轴偏移和非对称误差
- Dome-to-lens gap：模拟球罩与镜头前端的间隔，影响 dome 带来的附加误差
- Scene depth span：设定场景远端范围，用于改变 1 m 到远端之间的景深观察跨度
- Ray count：控制画面中显示的示意光线数量

### 4.4 Interaction

- Lens follow cursor：开启后，镜头会按设定强度跟随鼠标位置
- Follow amount：控制镜头跟随比例
- Reset Scene：恢复默认状态

### 4.5 Language Switch

页面头部提供语言下拉框，可在中文与英文界面之间切换。

## 5. 实时读数含义

- Subject distance：目标物距离镜头的当前距离
- Focus set distance：当前镜头位置对应的设定对焦距离
- Effective focal length：叠加镜头偏置后的等效焦距
- Thermal focus shift：温度变化引起的等效焦移
- Optical axis shift：Dome 倾斜等因素引起的像面偏移
- Dome edge penalty：Dome 边缘和离轴误差带来的附加惩罚项
- Sensor blur circle：传感器面上的模糊圈估计值
- Sharpness：根据总模糊量估算的清晰度等级
- Mode：当前交互状态，例如空闲、拖动目标、拖动镜头、镜头跟随

## 6. 交互方式

### 6.1 拖动目标物

当前版本中，目标物不会自动跟随鼠标，而是必须通过拖动改变位置。

### 6.2 拖动镜头

镜头通过拖动进行对焦设定调整。镜头位置变化不会进入无穷远奇点，而是被限制在有限的焦距范围内。

### 6.3 镜头跟随

启用 Lens follow cursor 后，镜头会在正常拖动之外，叠加一层由指针位置驱动的跟随行为。

## 7. 模型说明

这是一个为交互可视化而简化的近似模型，重点是趋势演示，不是精确光机仿真。

核心思路如下：

1. 使用薄透镜近似，根据等效焦距和物距计算像距。
2. 传感器平面固定，镜头拖动对应等效的对焦设定变化。
3. Dome 倾斜、间隙和离轴程度会引入额外像面偏移、边缘惩罚和不对称误差。
4. 温度项会改变传感器等效位置，从而造成热漂移失焦。
5. 预览窗口中的模糊强度按像素尺寸进行放大，便于观察宽角小焦距镜头中原本较细微的模糊变化。

## 8. 与代码对应的主要变量

在 `app.js` 中可以看到核心参数和读数来源：

- `LENS_SPEC`：镜头规格常量
- `defaults`：默认控制参数
- `computeOptics()`：核心光学近似计算
- `drawDomeCover()`：球罩前盖绘制
- `drawPreview()`：传感器预览绘制
- `drawFocusTravel()`：底部对焦行程条绘制
- `translations` / `setLanguage()`：中英文切换逻辑

## 9. 本地运行

本项目不依赖 Node.js，也不需要构建工具。

在项目根目录运行：

```bash
python3 -m http.server 8123
```

然后在浏览器打开：

```text
http://localhost:8123
```

## 11. 已知边界

- 不是严格的真实镜头设计软件，不能替代 Zemax、Code V 一类光学仿真工具
- Dome 影响目前是用于演示趋势的简化模型，尚未引入真实材料色散、厚度分布和完整折射面求解
- 传感器预览用于增强可读性，画面模糊强度做了可视化放大，不代表原始像面显示比例
